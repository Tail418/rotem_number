# 로뎀카페 주문번호 안내 시스템

포스트기에서 받은 주문번호를 TV 화면에 띄우고, 갤럭시탭(또는 다른 브라우저 기기)에서 번호 상태를 관리하는 로컬 시스템입니다.

## 구성

- [server.js](server.js) — Express + Socket.io 로컬 서버. 준비중/준비완료 상태를 메모리에 저장하고, 상태가 바뀔 때마다 연결된 모든 화면에 실시간으로 전달합니다.
- [public/tv.html](public/tv.html) — TV(노트북)에 띄우는 화면. 좌측 "준비중", 우측 "준비완료" 번호를 큰 글씨로 보여주고, 번호가 준비완료로 바뀌는 순간 "OO번 형제자매님 음료 준비되었습니다" 음성을 재생합니다.
- [public/admin.html](public/admin.html) — 갤럭시탭 등에서 접속하는 관리 화면. 번호 입력(등록), 완료 처리, 취소, 되돌리기, 전체 초기화 기능을 제공합니다.
- `package.json` — express, socket.io 의존성 정의

### 동작 방식

- 두 화면(TV / 관리)은 서버를 통해 실시간(WebSocket)으로 상태를 공유합니다.
- 같은 매장 와이파이 안에서만 동작하며, 외부 인터넷 연결은 필요 없습니다.
- 완료된 번호는 자동으로 지워지지 않고 계속 쌓이며, 관리 화면의 "전체 초기화" 버튼으로 한 번에 리셋합니다.
- 음성 안내는 TV 화면(노트북)의 브라우저 내장 TTS(Web Speech API)로 재생되며, 별도 음성 파일이나 인터넷 연결이 필요 없습니다.

## 사용 방법

### 1. 최초 설치

```bash
npm install
```

### 2. 서버 실행

매장 와이파이에 연결된 노트북에서 실행합니다.

```bash
node server.js
```

실행하면 콘솔에 아래와 같이 접속 주소가 출력됩니다.

```
로뎀카페 주문번호 시스템이 실행되었습니다.

  TV 화면    : http://localhost:3000/tv.html
  직원 입력  : http://192.168.x.x:3000/admin.html  (갤럭시탭에서 이 주소로 접속)
```

### 3. TV 화면 띄우기

- 노트북을 HDMI로 TV에 연결
- 노트북 브라우저에서 `http://localhost:3000/tv.html` 접속 후 전체화면(F11)

### 4. 갤럭시탭에서 관리 화면 접속

- 갤럭시탭이 노트북과 같은 매장 와이파이에 연결되어 있어야 합니다.
- 갤럭시탭 브라우저에서 `http://<노트북IP>:3000/admin.html` 접속 (IP는 서버 실행 시 콘솔에 출력된 주소 사용)
- 자주 쓴다면 갤럭시탭 홈 화면에 바로가기로 추가해두면 편리합니다.

### 5. 실제 사용 흐름

1. 직원이 관리 화면에서 번호를 입력하고 "등록" → TV의 "준비중"에 표시됨
2. 음료가 준비되면 관리 화면에서 해당 번호의 "완료" 버튼 클릭
3. 번호가 "준비완료"로 이동 + TV에서 음성 안내 자동 재생
4. 잘못 눌렀을 경우 "준비완료"의 "되돌리기"로 준비중으로 복귀 가능
5. 영업 마감 후 관리 화면 상단 "전체 초기화" 버튼으로 목록 전체 리셋

## 참고 사항

- 서버 상태는 메모리에만 저장되므로 서버를 재시작하면 초기화됩니다.
- 노트북 방화벽이 3000번 포트를 막고 있으면 갤럭시탭에서 접속이 안 될 수 있으니, 접속이 안 되면 방화벽 설정을 확인하세요.
- 노트북의 로컬 IP는 와이파이 재연결 시 바뀔 수 있으므로, 접속이 안 되면 서버를 껐다 켠 뒤 콘솔에 출력되는 새 주소를 확인하세요.
